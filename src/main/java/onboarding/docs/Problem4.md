# Problem 4   


## 👀 문제 확인하기  
- 우아한테크코스 1주차 과제 중 '문제 4'의 구현 기능과 구현 로직을 담은 문서입니다.
- 문제는 [여기](https://github.com/woowacourse-precourse/java-onboarding/blob/main/docs/PROBLEM4.md)에서 확인해 주세요.

---

## 🌟 구현 기능 목록  

### ✔ 입력 문자열 검증 기능
- 입력으로 들어온 문자열의 길이가 1~1000 사이인지 검증한다.

### ✔ 입력 문자열 분해 기능
- 문자열을 char형 배열로 분해한다.

### ✔ 알파벳 판단 기능
- 배열의 요소를 차례로 순회한다.
  - 요소가 알파벳인지 체크한다.
    - 알파벳이 아니라면 변환 작업 없이 그대로 붙인다.
    - 알파벳이라면 청개구리 사전의 규칙에 따라 변환을 진행한다.

### ✔ 청개구리 사전에 따른 변환 기능
- 대/소문자에 따라 변환을 진행한다.
  - 대문자면 'Z', 'A'를, 소문자면 'z', 'a'의 아스키코드 값을 활용하여 변환한다.

---

## 💡 구현 로직
~~~
1. 입력으로 들어온 문자열의 길이가 1~1000 사이인지 검증한다.

2. 입력으로 들어온 문자열을 char[]형 배열로 변환한다.

3. 배열 요소에 대해 순회를 진행한다.
  3-1. 알파벳인지 판단한다. 알파벳이 아니라면 그대로 붙인다.
  3-2. 알파벳이라면 변환 작업을 진행한다.

4. 알파벳에 대해 청개구리 사전의 규칙에 따라 변환 작업을 진행한다.
  4-1. 대문자의 경우 'Z'에서 (원본 요소 - 'A')를 뺀 아스키코드 값을 char형으로 변환한다.
  4-2. 소문자의 경우 'z'에서 (원본 요소 - 'a')를 뺀 아스키코드 값을 char형으로 변환한다.

5. 변환이 완료되면 결과값에 차례로 붙인 뒤, 최종적으로 리턴한다.
~~~
