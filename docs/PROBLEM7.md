## 🚀 기능 요구 사항

레벨 2의 팀 프로젝트 미션으로 SNS(Social Networking Service)를 만들고자 하는 팀이 있다. 팀에 속한 크루 중 평소 알고리즘에 관심이 많은 미스터코는 친구 추천 알고리즘을 구현하고자 아래와 같은 규칙을 세웠다.

- 사용자와 함께 아는 친구의 수 = 10점 
- 사용자의 타임 라인에 방문한 횟수 = 1점

사용자 아이디 user와 친구 관계 정보 friends, 사용자 타임 라인 방문 기록 visitors가 매개변수로 주어질 때, 미스터코의 친구 추천 규칙에 따라 점수가 가장 높은 순으로 정렬하여 최대 5명을 return 하도록 solution 메서드를 완성하라. 이때 추천 점수가 0점인 경우 추천하지 않으며, 추천 점수가 같은 경우는 이름순으로 정렬한다.

### 제한사항

- user는 길이가 1 이상 30 이하인 문자열이다.
- friends는 길이가 1 이상 10,000 이하인 리스트/배열이다.
- friends의 각 원소는 길이가 2인 리스트/배열로 [아이디 A, 아이디 B] 순으로 들어있다.
  - A와 B는 친구라는 의미이다.
  - 아이디는 길이가 1 이상 30 이하인 문자열이다.
- visitors는 길이가 0 이상 10,000 이하인 리스트/배열이다.
- 사용자 아이디는 알파벳 소문자로만 이루어져 있다.
- 동일한 친구 관계가 중복해서 주어지지 않는다.
- 추천할 친구가 없는 경우는 주어지지 않는다.

### 실행 결과 예시

| user | friends | visitors | result |
| --- | --- | --- | --- |
| "mrko" | [ ["donut", "andole"], ["donut", "jun"], ["donut", "mrko"], ["shakevan", "andole"], ["shakevan", "jun"], ["shakevan", "mrko"] ] | ["bedi", "bedi", "donut", "bedi", "shakevan"] | ["andole", "jun", "bedi"] |

#### 기능 구현 계획
문제 7번은 친구 추천 점수를 산출하여 사용자에게 추천할 친구를 반환해야 한다.  
추천 점수 산출에 앞서 사용자 아이디와 친구 관계 정보를 이용하여 모든 사용자의 친구 관계를 해시 테이블의 형태로 저장한다.  
친구 관계 테이블을 이용하여 추천 대상에 대해 사용자와 함께 아는 친구의 수를 알아낸다.  
알아낸 친구의 수를 통해 추천 대상들에 대해 추천 점수를 합산한다.  
이후 사용자의 타임 라인에 방문한 추천 대상의 방문 수를 집계하여 추천 점수를 갱신한다.  
추천 점수를 기반으로 최대 5명까지 추천할 대상을 선별하고 이미 사용자와 친구인 추천 대상은 선별 과정에서 제외한다.  
선별 과정을 거친 추천 대상들은 출력 시 점수순으로 정렬하고 만약 점수가 같다면 알파벳 순으로 정렬하여 최종적으로 추천 친구 목록을 반환한다.

문제 해결을 위해 필요한 기능은 다음과 같다.

User 클래스

1) isMyFriend 함수
- 입력: 사용자(추천받는 사용자와 추천되는 사용자 모두)의 친구 관계
- 기능: 사용자와 친구 관계인지 검사
- 출력: 친구 관계 여부

2) findLocation 함수
- 입력: 친구 관계 정보
- 기능: 친구 관계 정보에서 사용자 정보의 위치를 반환
- 출력: 사용자 정보의 위치

3) findFriends 함수
- 입력: 친구 관계 정보 리스트
- 기능: 주어진 친구 관계 정보 리스트에서 사용자와 친구인 사람을 탐색
- 출력: 사용자와 친구인 사람들 리스트

4) getName 함수
- 입력: -
- 기능: 사용자의 이름을 반환
- 출력: 사용자 이름

5) updateVisitScore함수
- 입력: -
- 기능: 추천 대상이 타임 라인 방문 시 정해진 점수만큼 추천 점수를 합산
- 출력: -

6) updateAcquaintanceScore 함수
- 입력: -
- 기능: 추천 대상이 사용자와 함께 아는 친구가 있는 경우 정해진 점수만큼 추천 점수를 합산
- 출력:-

7) getTotalScore 함수
- 입력: -
- 기능: 추천 점수 총합 반환
- 출력: 추천 점수 총합

8) getRelationship 함수
- 입력: -
- 기능: 사용자의 친구 관계 반환
- 출력: 사용자의 친구 관계

9) compareTo 함수
- 입력: User 클래스 타입 변수
- 기능: User 인스턴스 간의 추천 점수 대소 비교
- 출력: 추천 점수에 따른 두 객체의 차이

Relationship 클래스

1) getUiqueFriends 함수
- 입력: 친구 관계 정보
- 기능: 중복 없는 친구 리스트 생성
- 출력: -

2) makeRelation 함수
- 입력: 친구 관계 정보
- 기능: 중복 없는 친구 리스트를 활용하여 사용자 간의 관계 정보가 담긴 해시 테이블 생성
- 출력: 사용자 간의 관계 정보가 담긴 해시 테이블


Recommendation 클래스

1) addAcquaintanceScore 함수
- 입력: 추천 대상 이름
- 기능: 추천 대상의 사용자와 함께 아는 사람 수를 세어 추천 점수 합산
- 출력: -

2) checkAcquaintance 함수
- 입력: -
- 기능: 모든 추천 대상에 대해 사용자와 함께 아는 사람 수를 세어 추천 점수 합산
- 출력: -

3) isFriend 함수
- 입력: 추천 대상 이름
- 기능: 추천 대상이 사용자인지 판별
- 출력: 추천 대상 여부

4) isNewUser 함수
- 입력: 추천 대상 이름
- 기능: 추천 대상이 기존 추천 대상과 같은지 판별
- 출력: 기존 추천 대상과 동일 여부

5) getFriends함수
- 입력: -
- 기능: 추천 대상 목록 반환
- 출력: 추천 대상 목록

6) makeNewUser 함수
- 입력: 새로운 추천 대상
- 기능: 추천 대상 목록에 추가
- 출력: -

7) addVisitScore 함수
- 입력: 타임 라인을 방문한 추천 대상
- 기능: 방문한 추천 대상에게 미리 정의된 추천 점수 추가
- 출력: -

8) makeFriendScore 함수
- 입력: -
- 기능: 추천 점수 리스트 생성
- 출력: -
9) sortFriendScore 함수
- 입력: -
- 기능: 추천 점수 리스트를 추천 점수에 따라 정렬
- 출력: -

10) isNonZero 함수
- 입력: 추천 대상
- 기능: 추천 대상의 추천 점수가 0점인지 확인
- 출력: 추천 점수의 0점 여부

11) isAlreadyFriend 함수
- 입력: 추천 대상

- 출력: 추천 대상의 추천 여부를

13) exceedBounds 함수

- 출력: 추천 대상의 수 초과 여부

14) recommendFriends 함수
- 입력: -
- 기능: 추천 대상을 선정
- 출력: 추천 대상 리스트

15) checkBounds 함수
- 입력: 추천 대상 리스트

- 출력: 추천 대상 리스트
