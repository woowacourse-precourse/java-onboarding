## 문제 1

### input
- pobi, crong의 길이는 2

- 왼쪽과 오른쪽이 연속되지 않을 경우
- 숫자가 들어오지 않았을 경우
- 페이지의 숫자가 범위를 벗어났을 경우

#### 2. 각 자리 숫자를 더하는 기능
#### 3. 각 자리 숫자를 곱하는 기능

- 크루가 0이거나 10,001명 이상인 경우

#### 2. 닉네임 중복 확인할 Hash 생성

#### 4. 중복된 사용자의 이메일 오름차순 정렬
#### 5. 중복된 사용자의 이메일 출력

## 문제 7

### input
- 사용자의 아이디 user, 친구 관계 정보 friends, 사용자 타임라인 방문기록 visitors
- user의 길이는 1이상 30이하인 문자열
- friends의 길이는 1이상 10,000 이하인 리스트
- friends: [아이디A, 아이디B] : A와B는 친구
- visitors의 길이는 0이상 10,000이하인 리스트

### output
- 사용자와 함께 아는 친구의 수 +10
- 사용자의 타임라인에 방문한 횟수 +1
- 친구추천 점수가 가장 높은순 정렬하여 return
- 같을 시 이름순
- 추천 점수 0은 추천하지 않음
- 최대 5명

### 구현 기능 목록
#### 1. input 예외 확인 기능
- user의 길이가 1이상 30이하가 아닐경우
- friends의 길이가 1이상 10,000이하가 아닐 경우
- visitors의 길이가 0이상 10,000이하가 아닐 경우

#### 2. 사용자의 친구 list 생성
#### 2. Hash에 닉네임을 두개씩 나누어 저장
#### 3. 함께 아는 친구 점수 count
#### 4. 방문 횟수 점수 count
#### 5. Hash에서 사용자와 친구 제외
#### 6. 총 점수 count
#### 7. 추천친구 점수 내림차순 정렬
#### 8. 추천친구 이름 오름차순 정렬
